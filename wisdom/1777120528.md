# The Paradox of the Ship of Theseus in Software

Every time you patch a bug, refactor a function, or rename a variable, you're replacing a plank in the Ship of Theseus. After enough PRs, is it still the same codebase? Philosophers have debated this since 500 BC. Developers have been debating it since the first commit.

Fun fact: git blame was invented specifically so you could point at someone else and say "they put that plank there."
